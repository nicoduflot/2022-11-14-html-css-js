Introduction

Balises générales de contenu.

Généralement, tout les contenus de type texte sont a mettre dans des balise de type paragraphe. Il n'y a pas que la balise <p> </p>

Il y a deux type de balises qui contiennent du texte, celles avec un "display block" d'autres "inline".

  • Block :
    la balise prend tout l'espace en largeur, et ne peut pas contenir d'autre balise de texte de type block, sauf contre-indication en CSS.
    En revanche, elle peut contenir des balise de texte de type inline
  • Inline :
    ces balises entourent le texte, sans générer de marge, d'espace, sauf contre-indication en CSS.

Balises sur le texte

Il existe des balises inline pour modifier l'apparence du texte.

Par exemple, pour mettre simplement en gras, on utilise la balise <b>, mais si le texte mis en gras a une importance sur la pertinence du contenu niveau SEO, on utilisera la balise <strong>.
Il existe le même comportement pour l'italique : la balise <i> pour simplement l'afficher visuellement, la balise <em>, qui elle ajoute de l'emphase sur le contenu.

Autrefois, pour changer la couleur, la police d'un morceau de texte, on utilisait font, maintenant, on va plutôt utiliser une balise générale inline <span>.

Intégrer des images en HTML

Par défaut, la balise <img> est utilisée, elle a besoin d'attributs pour aller chercher et afficher l'image.
<img src="./images/image.png" />
src = la source de l'image.
Pour être dans les normes du W3C (world Wide Web Consortium, les garant des bonnes pratiques HTML), il faut ajouter l'attribut alt, qui défini un texte alternatif au cas ou l'image serait "cassée", le texte s'affichera en lieu et place de l'image
<img src="./images/image.png" alt="une image exemple" />

coucou

Avec HTML5, une balise sémantique est maintenant utilisée pour contenir les balises img, et permet aussi d'associé à cette image une légende qui sera reconnue comme légende de l'image par le SEO

Les formulaires

Les formulaires servent a récupérer des informations pour traitement, par exemple, un serveur php qui pourra les enregistrer en fichier, en bdd, ou les envoyer par mail

Un formulaire se déclare à l'aide de la balise <form>. Il possède des attributs, dont les plus courants et importants sont : method & action

method

méthode d'envoi des informations du formulaire, il en existe deux :

  • GET : les données sont envoyées associées à des noms de variables dans l'url de la page de traitement
  • POST : Les données sont transmises en variables dans l'entête HTTP.

Si la méthode n'est pas indiquée dans le formulaire, c'est GET qui sera utilisé par défaut

Action

Indique l'url de la page de traitement des données envoyées par le formulaire. Si action n'est pas précisé, le formulaire considère que c'est la page où il se trouve qui sera la page de traitement.
